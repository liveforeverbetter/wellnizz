## Health data you can use

| Data type | What you send | What the API returns |
| --- | --- | --- |
| [Biomarkers](/biomarkers) | Lab values as CSV, JSON, text, or supported PDFs | Consistent units, ranges that account for available age and sex context, calculated markers, trends, and ranked priorities |
| [Wearables](/wearables) | WHOOP OAuth, Google Health Connect bridge, or uploaded files | Sleep, recovery, heart rate, activity, and other trends the user has approved |
| [Genetics](/genetics) | DNA test exports or whole-genome VCF/VCF.GZ files uploaded directly to private storage | Health and trait context, ancestry, medication-response context, coverage, confidence, and job status |
| [Behavioral and lifestyle](/behavioral) | Supplements, medications, nutrition, symptoms, and goals | Structured context the API can compare with biomarkers, genetics, and wearables |

One data type is enough to produce a useful first result. Every response says
what data was used, what is missing, and which conclusions were measured,
calculated, or combined.

## Discover, trend, act

Every upload and result is listable, with no ID bookkeeping on your side:

- `GET /sources` and `GET /analyses`: everything a user has uploaded and computed
- `GET /analyses/{id}/recommendations`: prioritized, de-duplicated action items
- `POST /users/{user_id}/trends`: improving/worsening direction per marker across uploads
- `POST /analyses/{id}/rerun`: refresh results without re-upload
